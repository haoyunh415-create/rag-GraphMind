# 两分钟面试讲稿

## 0:00 - 0:20 项目定位

这个项目是一个本地可运行的 RAG 工程化平台，不是简单的 LLM 调用页面。它覆盖了知识文档上传、异步入库、混合检索、引用追踪、图谱增强、质量评估、延迟 Trace 和任务运营状态——七个阶段的完整闭环。

## 0:20 - 0:45 架构概述

前端是 Next.js，后端是 FastAPI。文档上传后先进入入库状态机，本地可以 inline/auto 模式跑，生产环境切换 Redis 持久队列。检索侧支持 SQLite 向量兜底，也能接入 Milvus、Elasticsearch、Neo4j。回答链路经过检索、重排、引用裁剪、生成和评估五个步骤，每个步骤都有 Trace 耗时记录。

## 0:45 - 1:15 演示流程

我会上传 3 份中文演示文档：第一份覆盖退货、发票和物流规则；第二份覆盖客服转人工和运营指标；第三份覆盖公司、产品、团队、负责人和故障依赖关系。然后按固定问题列表依次演示基础 RAG、列表回答和多跳关系推理，每次回答都展示引用来源和 Trace 面板。

## 1:15 - 1:40 可靠性设计

这个项目的重点是可解释和可回归验证。每个答案都有引用，Trace 面板能看到检索、重排、引用、评估各阶段的耗时和性能告警。入库任务有完整状态机，支持重试、取消、死信队列和队列健康监控。

## 1:40 - 2:00 质量评估

最后我会跑 Golden Eval 评测。它不是凭主观感受判断好坏，而是自动检查 Recall、MRR、引用精准率、拒答准确率、行为通过率、p95 延迟和性能告警。这样项目可以持续迭代，不是靠一次演示碰运气。

## 一句话总结

如果只能用一句话介绍：这是一个把 RAG 从"能回答"推进到"可解释、可评估、可运营"的完整工程化样例。
