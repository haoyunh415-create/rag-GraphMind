# Graph RAG: 下一代检索增强生成技术

## 概述

Graph RAG（Graph Retrieval-Augmented Generation）是一种将知识图谱与大型语言模型相结合的检索增强生成技术。与传统 RAG 仅依赖向量相似度搜索不同，Graph RAG 利用实体之间的结构化关系来提升检索的准确性和深度。

## 核心组件

### 1. 向量数据库 (Milvus)

Milvus 是一个开源的向量数据库，专为大规模向量相似度搜索而设计。它支持多种索引类型，包括 IVF_FLAT、HNSW 和 DiskANN，能够在十亿级数据集上实现毫秒级查询延迟。

Milvus 的核心架构包含以下组件：
- **Proxy**: 接收客户端请求，进行负载均衡
- **Root Coordinator**: 管理元数据和 DDL 操作
- **Data Coordinator**: 管理数据段的分配和持久化
- **Query Coordinator**: 管理查询节点的负载均衡
- **Index Coordinator**: 管理索引构建任务

### 2. 知识图谱 (Neo4j)

Neo4j 是一个图数据库，使用 Cypher 查询语言。在 Graph RAG 中，Neo4j 存储实体和它们之间的关系。例如，一篇关于机器学习的文档可能包含以下实体：
- **Python** 是 **编程语言**，用于机器学习
- **PyTorch** 是 **深度学习框架**，基于 Python
- **TensorFlow** 是 **深度学习框架**，由 Google 开发
- **Transformer** 是 **神经网络架构**，用于自然语言处理

Neo4j 通过 APOC 插件提供额外的图算法和数据处理能力，支持社区发现、中心性分析等高级图分析功能。

### 3. 全文搜索引擎 (Elasticsearch)

Elasticsearch 基于 Apache Lucene 构建，提供 BM25 算法进行关键词检索。在 Graph RAG 中，Elasticsearch 作为多路检索的一路，确保精确的关键词匹配不会被语义搜索所忽略。

Elasticsearch 的核心概念包括：
- **索引（Index）**: 文档的集合
- **分片（Shard）**: 索引的数据分区
- **副本（Replica）**: 分片的备份，提供高可用性
- **映射（Mapping）**: 定义文档字段的类型

## 检索流程

Graph RAG 的检索流程分为四个阶段：

### 阶段一：查询分解

用户输入的自然语言查询首先被分解为多个子查询。例如，查询「Python 在深度学习中的应用」可能被分解为：
1. Python 有哪些深度学习框架？
2. 这些框架各自的特点是什么？
3. 深度学习的典型应用场景有哪些？

### 阶段二：多路检索

系统并行执行三路检索：

1. **向量检索**：通过 embedding 模型将查询向量化，在 Milvus 中进行余弦相似度搜索
2. **关键词检索**：通过 Elasticsearch BM25 算法精确匹配关键词
3. **图谱检索**：在 Neo4j 中定位相关实体，并通过关系遍历找到关联内容

### 阶段三：融合排序

使用倒数排名融合（Reciprocal Rank Fusion, RRF）算法合并三路检索结果，去重后按综合得分排序，取 Top-K。

### 阶段四：答案生成

将排序后的文档片段和实体上下文注入 LLM 的 prompt，由 DeepSeek 或 GPT-4 等模型生成最终答案，同时附上引文标注。

## 缓存策略

系统使用 Redis 作为缓存层，对相同查询和上下文的答案进行缓存，TTL 设置为 1 小时。Redis 的 Pipeline 和 Pub/Sub 机制也被用于提高系统的并发处理能力。

## 部署架构

整个 Graph RAG 平台使用 Docker Compose 进行容器化部署，包含六个核心服务：

| 服务 | 用途 | 端口 |
|------|------|------|
| etcd | 分布式协调 | 2379 |
| Milvus | 向量存储与检索 | 19530 |
| Neo4j | 知识图谱存储 | 7687 |
| Elasticsearch | 全文检索 | 9200 |
| MinIO | 文档对象存储 | 9000 |
| Redis | 结果缓存 | 6379 |

所有服务通过 Docker 网络互联，后端使用 FastAPI 框架提供 RESTful API，前端使用 Next.js 14 构建用户界面。

## 应用场景

Graph RAG 适用于以下场景：

1. **企业知识库问答**：将内部文档、规范、报告构建为可查询的知识库
2. **科研文献检索**：对学术论文进行结构化检索和关联分析
3. **法律文档分析**：提取法律实体和案例之间的关系
4. **医疗知识管理**：构建疾病、症状、药物之间的知识图谱
5. **金融风险分析**：关联公司、人物、事件进行风险评估
